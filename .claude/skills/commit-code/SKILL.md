When writing a spec, always follow these steps:

1. **Create a commit message by stricly following the Conventional Commits specification**
2. **The body first pagraph should be a summary of is being achieved with the commit**
3. **The body second paragraph should be a bullet list of the main changes:** it should include references to the specific functions changed, including their declaration file name and line number
